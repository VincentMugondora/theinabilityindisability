# The Ability in Disability Trust (TADS) Website

## 🌍 Overview
This project contains the production-ready source code for the **The Ability in Disability Trust (TADS)** website. The site is a world-class, premium non-profit platform engineered to deliver outstanding performance across all network conditions (including 2G/3G rural networks) while maintaining the highest tiers of accessibility, SEO optimization, and an uplifting, trust-building user experience.

## 🚀 Core Mission & Engineering Objectives

1. **Uncompromising Performance**: Designed for slow networks first. Target metrics include First Contentful Paint (FCP) < 1.8s, Largest Contentful Paint (LCP) < 2.5s, Minimal JS Bundle (< 200KB) and Total Page Weight under 700KB.
2. **Accessibility & Inclusion**: Full ADA/WCAG compliance, utilizing semantic HTML, fully tabbable keyboard navigation, optimized contrast, and proper ARIA structuring.
3. **SEO Optimized**: Complete Open Graph, Canonical URLs, Semantic metadata formatting, Dynamic Sitemaps, and optimized for local searches (Zimbabwe/Harare).
4. **Modern, Sustainable Engineering**: Server-side rendered (SSR) & Static generated by default, strong typing, clean architectural layout, and easily scalable for a global reach.

## 💻 Tech Stack
- **Framework**: Next.js 16 (App Router paradigm)
- **Language**: TypeScript (Strict Mode)
- **Styling**: Tailwind CSS v4
- **Database/Auth**: Supabase (PostgreSQL, edge network integration)
- **Routing & Rendering**: React 19 (Server Components)
- **Forms & Validation**: React Hook Form with Zod
- **Icons**: Lucide React
- **Hosting Target**: Vercel

## 🛠️ Getting Started

Follow these instructions to run the application securely on your local machine.

### 1. Prerequisites 
Ensure that you have **Node.js (v20+)** installed.

### 2. Installation
Navigate directly into the Next.js project directory and install the necessary packages.

```bash
cd my-app
npm install
```

### 3. Environment Variables
To securely connect to API endpoints like the Supabase Database, locate the `.env.local.example` or equivalent schema, duplicate it, and rename it to `.env.local`. Fill it with your secret tokens.
```env
NEXT_PUBLIC_SUPABASE_URL=your-supabase-project-url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-supabase-anon-key
```

### 4. Run Development Server
```bash
npm run dev
```
Open [http://localhost:3000](http://localhost:3000) in your browser to observe the changes. The application utilizes `--webpack` hot reloading.

## 📐 Development & Design Guidelines

- **Always prioritize Server Components**: Use `"use client"` restrictively and purely for interactive UI states and forms.
- **Don't neglect loading states**: Incorporate `<Suspense>` bounds and lightweight skeleton loaders explicitly.
- **Avoid Heavy Libraries**: Only pull verified dependencies. Prefer Tailwind utility classes over pre-fabricated component libraries unless specifically required.
- **Reference Legacy Docs**: Check the `docs` and `templates` folders if you require visual cues or specific textual content. Modernize these concepts using the current stack layout.


## 🚀 Deployment
This repository is optimized automatically for **Vercel** serverless pipelines. To deploy, simply connect your Git repository to Vercel and it will parse `next build` appropriately. Ensure your production Environment Variables for Supabase and any outbound Email APIs are set in the Vercel dashboard prior to building.
